一.目标

这是一个kkb的接单平台，具体的内容就是实现学生导师的注册与使用功能

详细解说

这是一个总的流程图，当中包含各个方面的流程需要的技术

二.任务

我负责的任务为

①账号管理（管理员、用户）；

②登录/注册；

③个人中心

分析

1.账户管理

账户管理分为管理员和用户

管理员

管理员负责操作用户，相当于给角色赋予权限

用户

用户分为：游客，导师，学生

因此分为四个角色

2.登录与注册

登录

注册

3.个人中心

个人中心里有该用户的各种项目展示

三.实现

1.建表

理清各个角色之间的关系，然后就可以建表了

根据前面的概述，这里分为六个表。

用户表：存储用户详细信息

角色表：存储相应角色信息

权限表：存储权限，比如学生权限，导师权限，管理员权限，然后根据相应权限展示相应页面

用户与角色关联表：相当于是一个中间表，用户对应相应角色

角色与权限关联表：也是一个中间表，角色对应相应权限

项目表：个人中心里面有展示用户的相应项目，所以需要一个项目表

这是建表的格式，需要频繁的查询某个字段的话就建立一个索引。在这样的话就能提高效率

主从复制

http://www.macrozheng.com/#/reference/mysql_master_slave

概念：

主从复制，是用来建立一个和主数据库完全一样的数据库环境，称为从数据库，主数据库一般是准实时的业务数据库。

优点：

1–在从服务器可以执行查询工作(即我们常说的读功能)，降低主服务器压力;（主库写，从库读，降压）
2–在从主服务器进行备份，避免备份期间影响主服务器服务;（确保数据安全）
3–当主服务器出现问题时，可以切换到从服务器。（提升性能）

原理

1.数据库有个bin-log二进制文件，记录了所有sql语句。

2.我们的目标就是把主数据库的bin-log文件的sql语句复制过来。

3.让其在从数据的relay-log重做日志文件中再执行一次这些sql语句即可

五.读书的内容

我选读的书为java核心知识要点。这边说一下自己看的JVM把

JVM

概念

JVM 是可运行 Java 代码的假想计算机 ，包括一套字节码指令集、一组寄存器、一个栈、 一个垃圾回收，堆 和 一个存储方法域。JVM 是运行在操作系统之上的，它与硬件没有直接 的交互。

编译过程

① Java 源文件—->编译器—->字节码文件

② 字节码文件—->JVM—->机器码

每一种平台的解释器是不同的，但是实现的虚拟机是相同的，这也就是 Java 为什么能够 跨平台的原因了 ，当一个程序从开始运行，这时虚拟机就开始实例化了，多个程序启动就会 存在多个虚拟机实例。程序退出或者关闭，则虚拟机实例消亡，多个虚拟机实例之间数据不 能共享。

GC

JVM虚拟机垃圾回收算法

如何确定垃圾

1.引用计数法

对象如果没有任何与之关 联的引用，即他们的引用计数都不为 0，则说明对象不太可能再被用到，那么这个对象就是可回收 对象。

2.可达性分析

在“GC roots”和一个对象之间没有可达路径，则称该对象是不可达的

不可达对象变为可回收对象至少要经过两次标记 过程。

什么叫做可达：https://www.jianshu.com/p/854ef1afe763

3.复制算法（copying）

为了解决 Mark-Sweep 算法内存碎片化的缺陷而被提出的算法。按内存容量将内存划分为等大小 的两块。每次只使用其中一块，当这一块内存满后将尚存活的对象复制到另一块上去，把已使用 的内存清掉
